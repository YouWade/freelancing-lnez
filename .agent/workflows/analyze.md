# Analyze Code Skill

## 功能描述

分析指定的程式碼檔案或目錄，提供程式碼品質報告。

## 使用方式

- `/analyze` - 分析當前開啟的檔案
- `/analyze <file_path>` - 分析指定檔案
- `/analyze <directory>` - 分析整個目錄

## 分析項目

### 1. 程式碼結構

- 檔案組織
- 模組化程度
- 依賴關係

### 2. 程式碼品質

- 可讀性評估
- 命名規範
- 註解完整度
- 複雜度分析

### 3. 最佳實踐

- 設計模式使用
- 錯誤處理
- 效能考量
- 安全性檢查

### 4. 改進建議

- 重構建議
- 優化方向
- 潛在問題

## 執行指令

當使用者呼叫此 skill 時：

1. 讀取指定的檔案或目錄
2. 分析程式碼結構和品質
3. 提供清晰的報告，包含：
   - ✅ 做得好的地方
   - ⚠️ 需要注意的地方
   - 💡 改進建議
4. 優先考量實用性，避免過度理論化
